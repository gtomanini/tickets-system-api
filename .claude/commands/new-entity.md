Create a new JPA entity for the specified domain: $ARGUMENTS

Follow the patterns defined in CLAUDE.md exactly. Execute the steps below in order:

## 1. Determine the entity type

From `$ARGUMENTS`, identify:
- **Name** of the entity (e.g. `Discount`, `EventTag`)
- **ID type**: use `UUIDIdEntity` for transactional data linked to users; use `AutoIncrementIdEntity` for reference/configuration data
- **Fields** required (if not specified, ask the user)
- **Relationships** with existing entities

## 2. Create the entity in `models/`

```java
@Entity
@Table(name = "<table_name>")
@AllArgsConstructor
@NoArgsConstructor
@Builder
@Getter
@Setter
public class <EntityName> extends <AutoIncrementIdEntity|UUIDIdEntity> {
    // fields here
}
```

Rules:
- `@ManyToOne` and `@OneToMany` relationships must have an explicit `fetch` type
- Use `FetchType.LAZY` by default; `EAGER` only when justified
- Long text fields: `@Column(columnDefinition = "TEXT")`
- Monetary values: `BigDecimal` (never `double` for money)
- Dates/times: `LocalDateTime`

## 3. Create the Repository in `repositories/`

```java
public interface <Name>Repository extends JpaRepository<<Name>, <IdType>> {
}
```

Add `JpaSpecificationExecutor<<Name>>` if the entity will need filtered searches.

## 4. Verify relationships

For each relationship with existing entities:
- Add the inverse side on the related entity if needed (`@OneToMany(mappedBy=...)`)
- Check if `cascade = CascadeType.ALL` is needed on `@OneToMany`

## 5. Document the generated table

Describe the SQL table that Hibernate will auto-generate (`ddl-auto=update`):
```sql
-- Table auto-generated by Hibernate
CREATE TABLE <table_name> (
  id ...,
  created_at DATETIME,
  updated_at DATETIME,
  deleted BOOLEAN DEFAULT FALSE,
  -- other fields
);
```

List:
- Entity file created
